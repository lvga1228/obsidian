# Supply Chain Stock Flow Analysis Framework

## Overview
Analyzes warehouse stock turnover efficiency by calculating weekly inflow/outflow ratios from stock balance data. Useful for comparing different business units (BUs), identifying operational improvements, and monitoring inventory health.

## Quick Start

### Typical Use Case
```python
# User provides 4 CSV files: Foods and HPC stock/outflow data
# Question: "Foods仓库的进出量占仓库的总库存比例和HPC的比例差别大吗?"

# Analysis steps:
1. Read and clean the 4 CSV files
2. Apply stock balance equation: inflow = next_week_stock + outflow - current_stock
3. Calculate turnover ratios at two levels: aggregated (week-BU) and granular (plant-week)
4. Compare Foods vs HPC using statistical tests
5. Generate visualizations and comprehensive report
6. Organize outputs in timestamped folder
```

### Key Decisions to Make
1. **Analysis level**: Aggregated (business reporting) vs Granular (operational improvement)
2. **Outlier handling**: Filter extreme values (>5 ratio) or use weighted/median metrics
3. **Time period**: Full history vs recent year (e.g., 2025)
4. **Output format**: Comprehensive report + CSV data + PNG visualizations

### Expected Outputs
- **Statistical summary**: Weighted turnover ratios, median values, sample sizes
- **Visualizations**: Distribution plots, time series, box plots, plant rankings
- **Business insights**: Turnover differences, improvement trends, data quality issues
- **Organized files**: All outputs in `分析主题_YYYYMMDD_HHMM` folder

## Core Concepts

### Stock Balance Equation
```
当周进量 (inflow) = 下一周的仓储总量 + 当周出量 - 当周库存
```
This formula derives inflow from sequential stock levels and outflow data when direct inflow data is unavailable.

### Key Metrics
1. **Outflow Ratio** = Outflow / Stock
2. **Inflow Ratio** = Inflow / Stock  
3. **Turnover Ratio** = (Inflow + Outflow) / Stock = Outflow Ratio + Inflow Ratio

## Analysis Workflow

### Step 1: Data Preparation
```python
import pandas as pd

# Read stock and outflow data
stock_df = pd.read_csv(stock_path)
out_df = pd.read_csv(out_path)

# Ensure consistent data types
stock_df["week_monday"] = pd.to_datetime(stock_df["week_monday"])
out_df["week_monday"] = pd.to_datetime(out_df["week_monday"])
stock_df["plant_code"] = stock_df["plant_code"].astype(str)
out_df["plant_code"] = out_df["plant_code"].astype(str)
```

### Step 2: Data Cleaning
```python
# Filter by BU if needed
if bu_filter is not None:
    stock_df = stock_df[stock_df["BU"].isin(bu_filter)].copy()
    out_df = out_df[out_df["BU"].isin(bu_filter)].copy()

# Handle duplicates: average values for same plant-week-BU
stock_agg = stock_df.groupby(["week_monday", "BU", "plant_code"], as_index=False)["stock_qty"].mean()
out_agg = out_df.groupby(["week_monday", "BU", "plant_code"], as_index=False)["actual_stock_out"].mean()
```

### Step 3: Temporal Alignment
```python
# Sort by BU, plant, week for shift operations
stock_agg = stock_agg.sort_values(["BU", "plant_code", "week_monday"]).copy()

# Find next week's stock
stock_agg["next_week_monday"] = stock_agg.groupby(["BU", "plant_code"])["week_monday"].shift(-1)
stock_agg["next_stock_qty"] = stock_agg.groupby(["BU", "plant_code"])["stock_qty"].shift(-1)

# Keep only consecutive weeks (7-day difference)
stock_agg = stock_agg[
    (stock_agg["next_week_monday"] - stock_agg["week_monday"]).dt.days == 7
].copy()
```

### Step 4: Calculate Inflow
```python
# Merge stock and outflow data
merged = stock_agg.merge(
    out_agg,
    on=["week_monday", "BU", "plant_code"],
    how="inner"
)

# Calculate inflow using stock balance equation
merged["stock_in_calc"] = (
    merged["next_stock_qty"]
    + merged["actual_stock_out"]
    - merged["stock_qty"]
)

# Remove zero/negative stock records (can't calculate ratios)
merged = merged[merged["stock_qty"] > 0].copy()
```

### Step 5: Calculate Ratios
```python
# Calculate individual ratios
merged["stock_out_ratio"] = merged["actual_stock_out"] / merged["stock_qty"]
merged["stock_in_ratio"] = merged["stock_in_calc"] / merged["stock_qty"]

# Optional: total turnover ratio
merged["turnover_ratio"] = merged["stock_out_ratio"] + merged["stock_in_ratio"]
```

### Step 6: Statistical Summary
```python
def summarize_ratio(df, period_name):
    summary = []
    
    for dataset, sdf in df.groupby("dataset"):  # or "BU"
        summary.append({
            "period": period_name,
            "dataset": dataset,
            "row_count": len(sdf),
            "plant_count": sdf["plant_code"].nunique(),
            "week_count": sdf["week_monday"].nunique(),
            
            # Weighted ratios: sum(flow) / sum(stock) - reflects overall business scale
            "stock_out_ratio_weighted": sdf["actual_stock_out"].sum() / sdf["stock_qty"].sum(),
            "stock_in_ratio_weighted": sdf["stock_in_calc"].sum() / sdf["stock_qty"].sum(),
            "turnover_ratio_weighted": (sdf["actual_stock_out"].sum() + sdf["stock_in_calc"].sum()) / sdf["stock_qty"].sum(),
            
            # Median ratios: typical plant-week performance (robust to outliers)
            "stock_out_ratio_median": sdf["stock_out_ratio"].median(),
            "stock_in_ratio_median": sdf["stock_in_ratio"].median(),
            "turnover_ratio_median": sdf["turnover_ratio"].median(),
            
            # Mean ratios: sensitive to outliers (useful for data quality checks)
            "stock_out_ratio_mean": sdf["stock_out_ratio"].mean(),
            "stock_in_ratio_mean": sdf["stock_in_ratio"].mean(),
            "turnover_ratio_mean": sdf["turnover_ratio"].mean(),
            
            # Data quality indicators
            "negative_stock_in_pct": (sdf["stock_in_calc"] < 0).mean(),
            "extreme_ratio_pct": ((sdf["stock_out_ratio"] > 10) | (sdf["stock_in_ratio"] > 10)).mean()
        })
    
    return pd.DataFrame(summary)
```

## Interpretation Guidelines

### Turnover Ratio Interpretation
- **< 0.5**: Low turnover - inventory moves slowly relative to stock levels
- **0.5-1.0**: Moderate turnover - typical range for many warehouses  
- **1.0-2.0**: High turnover - inventory moves quickly
- **> 2.0**: Very high turnover - may indicate just-in-time operations or data issues

### Data Quality Checks
1. **Negative inflow ratio**: Should be < 5%. Higher indicates data quality issues
2. **Extreme ratios (>5)**: Often caused by very small stock quantities - check for data errors
3. **Extreme ratios (>10)**: Very likely data errors - filter out or investigate
4. **Missing consecutive weeks**: Gaps in temporal alignment reduce sample size
5. **Mean vs Median discrepancy**: If mean >> median, data has positive skew from outliers

### Comparative Analysis
1. **Between BUs**: Identify which business units have higher/lower turnover
2. **Over time**: Track turnover trends to measure improvement initiatives
3. **Between plants**: Identify high/low performing facilities within same BU

## Common Pitfalls

### 1. Temporal Misalignment
- Ensure week definitions are consistent across datasets
- Verify that `shift(-1)` correctly captures next week's data
- Handle holiday weeks or missing data appropriately

### 2. Outlier Handling
- Small stock values create extreme ratios - consider winsorizing or filtering
- Weighted ratios (sum(flow)/sum(stock)) are more stable than mean ratios
- Always report median alongside mean to identify skewness

### 3. Business Context
- Different products/BUs may have inherently different turnover expectations
- Seasonal patterns affect weekly ratios - consider longer time windows
- Check with business users if negative inflow makes sense (returns, adjustments)

7. **EDA for Stock Out Data Insights**
   - **Environment verification**: Always test Python package imports before full analysis execution. Use `/opt/anaconda3/envs/code_mbp/bin/python -c "import pandas"` to validate environment setup.
   - **Negative outflow handling**: For stock outflow data, negative values (typically <1% of records) may represent returns or adjustments. Three recommended approaches:
     1. Treat as zero (assuming minor adjustments)
     2. Keep but flag for separate analysis
     3. Remove if negligible impact on overall statistics
   - **Factory-level Pareto analysis**: Typically 20% of factories contribute 80% of total outflow. Identify high/low performing factories using:
     - Average outflow volume
     - Outflow volatility (coefficient of variation)
     - Consistency over time
   - **Business granularity clarification**: Always explicitly state what each row represents (e.g., "single factory-week outflow record") in analysis reports.
   - **Visualization priorities for outflow data**:
     1. Time series trends (weekly total, top factories)
     2. Distribution histograms (with log scale for long-tailed data)
     3. Factory comparison box plots
     4. Heatmaps for recent period patterns
     5. Cumulative outflow by factory
   - **Data health scoring**: Develop consistent scoring system (0-10) based on: missing values, duplicates, temporal continuity, outlier prevalence, and logical consistency.
